## 4. 감속 문제 해결 과정

| 버전 | 감속 RMSE z | 감속 최대 Dz | 원인/수정 |
|------|-----------|-------------|----------|
| 수정 전 | 8.15 | 22.4 | INDI G가 추력 23% 과대평가 |
| + 전진비 수정 (3-1) | **1.45** | **5.0** | G 행렬에 advance ratio 반영 |
| + RTK EKF | **0.83** | - | EKF 스무딩이 INDI 안정화 |

근본 원인: 감속 중 드론이 30도 이상 틸트 -> V_axial이 커지면서 실제 추력이 감소하는데,
기존 INDI는 이를 무시하고 추력을 과대평가 -> 모터 under-command -> 고도 추락.

수정: dT/dn = k_T * n * (1 + fac) (전진비 반영, 기존 2 * k_T * n에서 변경).

잔존 이슈: MC 30회 중 3%에서 20m+ 발생 (INDI 유한차분 각가속도 추정의 구조적 한계).
RTK EKF 스무딩 + 폴백 제어기로 충분히 대응 가능.

## 5. 제어기별 종합 평가

### LQR (ScheduledLQR)
- 장점: 계산 빠름 (12초), 순항 고도 안정 (0.26m)
- 단점: 속도 추종 불가 (RMSE vx=16.8, 70 m/s에 못 미침), 호버 진입 불안정 (Dz 9.7m)
- 역할: 폴백 전용

### NMPC (CasADi+IPOPT)
- 장점: 감속 최강 (0.50m), 안정적, 발산 없음
- 단점: 느림 (165초, 실시간 불가), 순항 고도 오프셋 (1.76m), 센서 노이즈에 취약
- 역할: 단독 사용 부적합. Hybrid 내부 플래너(VirtualNMPC)로서만 유효.

### Hybrid (ProperHybrid = VirtualNMPC + INDI)
- 장점: 5/6 구간 1위, 돌풍 면역 (Dz=0.15m), 완벽 수렴, 속도 추종 최강 (1.88 m/s)
- 단점: 감속이 여전히 최약 구간 (1.45m 참값, 0.83m RTK EKF)
- 역할: **주 제어기. RTK EKF + 폴백과 함께 실기 권장.**

## 6. 최종 결론

### 확정 사항
1. **Hybrid + RTK EKF = RMSE z 1.670m**, 전 구간 최강
2. **RTK GPS 필수** — EKF 스무딩이 INDI 감속 문제를 추가 억제
3. **NMPC 단독 부적합** — 느리고, 센서 노이즈에 취약하고, setpoint 한계
4. **LQR = 폴백 전용** — 속도 못 쫓지만 안 터짐
5. **MC 통계적 우위 확인** — Hybrid 최악 < LQR 최선 (10회 기준)
