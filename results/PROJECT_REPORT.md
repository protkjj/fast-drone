## 1.0 한 줄 결론

> **"NMPC 만능"이 아니라, 상황/목표에 따라 최적 제어기가 다르다.
> 단, RTK GPS 확보 시 분리 하이브리드가 전 시나리오 1위. 표준GPS에서는 LQR로 역전.**

순항 고도는 NMPC/LQR스케줄, 가속/큰 과도는 NMPC, 돌풍 외란은 LQR/피드백 계열이 우수하다.
나이브한 NMPC+INDI 결합은 이중 보정으로 나빠지지만, **인터페이스를 분리한 하이브리드는 전 조건에서 최강**이 된다.
즉 알고리즘 선택만큼 **아키텍처(어떻게 연결하는가)**가 성능을 좌우한다.

## 1.1 프로젝트 개요

| 항목 | 확정 내용 |
|------|----------|
| 역할 | 고속 정찰(ISR) |
| 형상 | 축대칭 미사일형 동체 + 쿼드콥터 추진(z추력 로터 4개) |

| 도구 | CasADi(동역학/자동미분), acados(실시간 NMPC) |
| 좌표계 | NWU(관성) + FRD(동체). PX4 NED와는 Rx(180도) 변환 |

## 1.2 플랜트 모델

- CasADi 6-DOF, 상태 17차원: [위치3, 속도3, 쿼터니언4, 각속도3, 로터4]
- 동체 공력(수직력/축력/모멘트) + 4로터(전진비 J 의존 추력) + 자이로/반토크
- V=0 특이점 대수적 소거 -> acados 호환
- CasADi 하나로 플랜트 시뮬 + NMPC 예측모델 재사용
- test_plant.py 6개 물리 검증 통과
- 파라미터: C_A0=0.5->0.12(유선형) 조정 -> 85 m/s 트림 가능, 여유 48%
- 트림 모터 사용률: 호버 32%, 85 m/s 52% (적정 T/W=5~6, 현재 9.9은 과도)

## 1.3 수정된 버그 (결과 신뢰도)

| # | 내용 | 영향 |
|---|------|------|
| LQR 발산 | Error-State 17D->14D 축소 + Q_L 부호 수정 | 이전 "LQR 발산"은 이 버그 |
| acados NaN | yref 런타임 파라미터화 + 트림 warm start + 해 시프트 | 전 속도 수렴 |
| 1-1 | 폴백 NaN 검사가 채터링 가드 뒤에 있어 1초간 비활성화 | 안전 |
| 1-2 | ESKF가 RTK R값 무시 (항상 sigma=1.5m로 동작) | RTK 결과 전체 |
| 1-3 | NMPC/VirtualNMPC warm start(w0) 리셋 누락 | MC 독립성 |
| 2-2 | VirtualNMPC T_ref 미전달 | 비용함수 정확도 |
| 3-1 | INDI G 행렬에 전진비(advance ratio) 미반영 | 감속 폭발 -82% 해결 |

특히 1-2 수정으로 RTK+Hybrid 성능이 -55% 개선, 3-1 수정으로 감속 RMSE z 8.15->1.45.

## 1.6 감속 문제 해결 과정

| 버전 | 감속 RMSE z | 감속 최대 Dz | 원인/수정 |
|------|-----------|-------------|----------|
| 수정 전 | 8.15 | 22.4 | INDI G가 추력 23% 과대평가 |
| + 전진비 수정 (3-1) | 1.45 | 5.0 | dT/dn = k_T*n*(1+fac) |
| + RTK EKF | **0.83** | - | EKF 스무딩이 INDI 안정화 |

잔존: MC 30회 중 3%에서 20m+ (INDI 유한차분 한계). 폴백 제어기로 대응.

## 1.9 연구 서사

1. **동기**: 미사일형+쿼드로 300km/h 정찰. 고속은 공력 지배/비선형이라 제어 난제.
2. **가설**: 고속에서 고전제어 붕괴, NMPC 필요.
3. **발견**: 가설은 부분적으로만 맞음. LQR도 스케줄링하면 넓은 영역 커버. NMPC는 속도/과도에 강하나 튜닝 민감/예측불가 외란 약함. 나이브 하이브리드는 이중보정으로 실패, 인터페이스 분리 시 전 조건 최강.
4. **감속 문제 발견/해결**: 통합 미션에서 INDI G 전진비 미반영 발견 (23% 추력 과대평가). 수정으로 감속 RMSE 82% 개선. RTK EKF 스무딩으로 추가 43% 개선.
5. **결론**: 단일 제어기 우월이 아니라 상황별 최적이 다름. 예측+피드백 결합이 이상적이나 **설계(인터페이스 분리)가 관건**.
6. **ESKF 검증**: RTK면 순위 유지, 표준GPS면 역전. 센서 품질이 제어기 선택을 좌우.

# 파트 2 -- 제어기 이론

## 2.1 PID
- 원리: u = Kp*e + Ki*integral(e) + Kd*de/dt
- 장점: 단순, 적분기로 정상상태오차 제거, 모델 불필요
- 단점: 단일 동작점 (고속 6배 악화), 캐스케이드 지연, 커플링 무시

## 2.2 LQR
- 원리: J = integral(x'Qx + u'Ru) 최소화 -> u = -Kx (리카티)
- 장점: 최적성, 커플링 처리, 즉각 피드백 (돌풍 우수)
- 단점: 선형화 근처만, 적분기 없어 정상상태오차, 제약 없음

## 2.3 NMPC
- 원리: 비선형 예측 + 제약 하 비용 최소화, 첫 항만 적용 (Receding Horizon)
- 장점: 예측 (선제대응), 제약 처리, 비선형 모델
- 단점: 연산 부담, 모델 의존, 가중치 민감

## 2.4 INDI
- 원리: du = G_inv * (x_dot_des - x_dot_meas), 센서로 증분만 계산
- 장점: 모델의존 최소, 즉각 반응 (1kHz)
- 단점: 센서 노이즈, 제약 없음, 배치 위치 결정적

## 2.5 분리 하이브리드 (ProperHybrid)
- 구조: VirtualNMPC(13D) -> [T_cmd, omega_dot_des] -> INDI -> [n1..n4]
- NMPC는 "뭘 할지", INDI는 "어떻게 할지" (역할 분리)
- 나이브 결합 함정: 이중 보정 3.5배 악화 -> 인터페이스 분리로 1.7배 개선
- INDI G에 전진비 반영 (3-1): 감속 추력 과대평가 23% 해소
- 전 조건 최강의 핵심: 아키텍처가 알고리즘만큼 중요

## 2.6 한눈 비교

| | PID | LQR | NMPC | INDI | Hybrid |
|--|-----|-----|------|------|--------|
| 유형 | 반응 | 반응(최적) | 예측 | 반응(센서) | 예측+센서 |
| 모델 의존 | 없음 | 선형 | 높음 | 매우낮음 | 중간 |
| 제약 처리 | X | X | O | X | O(NMPC) |
| 연산 | 매우낮음 | 낮음 | 높음 | 낮음 | 높음 |
| 통합미션 z | 4.44 | - | 2.80 | - | **1.75** |
